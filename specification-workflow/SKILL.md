# Specification Workflow

## Purpose

Create high-quality Markdown documentation from mixed inputs through deep interview, explicit planning, drafting, and review loops.

## Inputs

- Source files, links, notes, screenshots, or rough requirements.
- Target audience and intended decision/use.
- Required output location and format constraints.

## Workflow

1. Inventory the input materials.
2. Run a deep interview to clarify users, scope, constraints, examples, edge cases, and acceptance criteria.
3. Produce a documentation plan with checkbox sections.
4. Draft Markdown documentation.
5. Run a review cycle for correctness, completeness, terminology, and actionability.
6. Apply review feedback.
7. Optionally prepare worklog entry after approval.

## Documentation Rules

- Markdown is the default output.
- Separate facts, assumptions, open questions, and decisions.
- Keep source references traceable.
- Do not fabricate requirements or system behavior.

## Outputs

- Interview notes.
- Documentation plan.
- Markdown specification.
- Review notes and optional worklog proposal.
